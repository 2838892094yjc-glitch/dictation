## 1. 产品概述

### 1.1 产品背景

英语听写是语言学习的基础环节，但传统听写方式存在以下痛点：
- **家长时间受限**：无法随时为孩子报听写
- **单词录入繁琐**：手动输入单词表费时费力
- **批改耗时**：对照答案逐一批改效率低
- **缺乏灵活性**：无法自定义听写模式和顺序

### 1.2 产品定位

一款面向学生和家长/教师的智能英语听写工具，通过 OCR + TTS 技术，实现「拍照录入-语音播报-拍照批改」的全流程自动化。

### 1.3 目标用户

| 用户群体 | 年龄/身份 | 核心需求 | 使用场景 |
|---------|----------|---------|---------|
| 小学生 | 6-12岁 | 完成课内单词听写作业 | 课后复习、考前准备 |
| 中学生 | 13-18岁 | 中考/高考词汇背诵 | 自主练习、查漏补缺 |
| 家长 | 25-45岁 | 替代人工报听写、自动批改 | 家庭作业辅导 |
| 英语自学者 | 18-35岁 | 灵活自定义词汇学习 | 碎片化时间学习 |

## 2. 竞品分析

### 2.1 现有开源项目

| 项目 | 技术栈 | 核心功能 | 缺点 |
|-----|-------|---------|-----|
| **[TypeWords](https://github.com/zyronon/TypeWords)** | Nuxt.js | 多模式练习、记忆曲线、丰富词库 | 无拍照功能、无OCR |
| **[word-dictation-practice](https://github.com/zxkane/word-dictation-practice)** | Next.js + MUI | 基础听写练习 | 功能简单、无语音播报 |
| **[Words](https://github.com/berk76/words)** | Java | 分类管理、发音下载、测试统计 | 桌面应用、无OCR |
| **[PicToSpeech](https://github.com/zaker-amin/PicToSpeech)** | Android | OCR + TTS + Gemini | 专注视障辅助、非听写场景 |

### 2.2 商业产品

| 产品 | 厂商 | 核心功能 | 价格 |
|-----|-----|---------|-----|
| **新东方单词听写** | 新东方 | OCR录题、语音播报、拍照批改 | API付费 |
| **扇贝单词** | 扇贝 | 智能复习、AI精简规划 | 会员制 |
| **每日英语听力** | 欧路 | 听力训练、取词翻译 | 免费+内购 |
| **AI Grammar** | HelloTalk | 作文拍照批改 | 免费+内购 |

### 2.3 竞品结论

**市场空白**：
- 开源领域缺乏完整功能的「拍照录题+语音听写+拍照批改」一体化方案
- 商业产品多为综合学习平台，专注听写场景的工具较少
- 现有开源项目多为纯软件方案，缺乏OCR和手写识别能力

## 3. 功能需求

### 3.1 核心功能模块

```
┌─────────────────────────────────────────────────────────────────┐
│                     自动英语听写软件                              │
├─────────────────┬─────────────────┬─────────────────────────────┤
│    📷 录题模块    │   🔊 听写模块    │       ✅ 批改模块           │
├─────────────────┼─────────────────┼─────────────────────────────┤
│ • 拍照上传单词表 │ • 语音播报单词   │ • 拍照上传听写纸            │
│ • 相册选择图片   │ • 双语切换模式   │ • 手写文字识别              │
│ • 图片裁剪/旋转  │ • 语速调节       │ • 智能比对批改              │
│ • OCR文字识别   │ • 重复次数设置   │ • 错误标记与统计            │
│ • 单词自动提取   │ • 间隔时间设置   │ • 错题本生成                │
│ • 手动编辑校正   │ • 随机/顺序模式  │ • 成绩报告导出              │
└─────────────────┴─────────────────┴─────────────────────────────┘
```

### 3.2 功能详细说明

#### 3.2.1 录题模块

| 功能 | 说明 | 优先级 |
|-----|-----|-------|
| 拍照上传 | 调用相机拍摄单词表照片 | P0 |
| 相册导入 | 从手机相册选择图片 | P0 |
| 图像预处理 | 自动纠偏、增强对比度、去噪 | P1 |
| 手动裁剪 | 框选单词表区域 | P0 |
| OCR识别 | 识别图片中的英文/中文文字 | P0 |
| 单词提取 | 自动提取单词对（英+中） | P0 |
| 表格识别 | 识别表格格式的单词表 | P1 |
| 批量编辑 | 增删改查识别结果 | P1 |
| 词库保存 | 保存到本地词库，支持命名 | P0 |
| 词库导入/导出 | JSON/Excel/TXT 格式 | P2 |

**支持的图片格式**: PNG、JPEG、BMP  
**图片大小限制**: ≤ 10MB  
**识别语言**: 英文、简体中文、繁体中文

#### 3.2.2 听写模块

| 功能 | 说明 | 优先级 |
|-----|-----|-------|
| 英译中模式 | 播报英文，要求写出中文释义 | P0 |
| 中译英模式 | 播报中文，要求写出英文单词 | P0 |
| 拼写模式 | 播报英文+释义，要求完整拼写 | P0 |
| 语音选择 | 美式/英式发音切换 | P1 |
| 语速调节 | 0.5x - 2.0x 速度调节 | P1 |
| 重复次数 | 每个单词重复 1-3 次 | P1 |
| 间隔时间 | 单词间间隔 3-10 秒可调 | P1 |
| 播报顺序 | 顺序/随机/智能（易错优先） | P1 |
| 进度显示 | 当前进度 3/20 | P0 |
| 暂停/继续 | 听写过程可控 | P0 |
| 上一词/下一词 | 手动切换 | P1 |
| 拼写提示 | 显示首字母或音节数 | P2 |

#### 3.2.3 批改模块

| 功能 | 说明 | 优先级 |
|-----|-----|-------|
| 拍照上传答案 | 拍摄听写纸照片 | P0 |
| 手写识别 | OCR识别手写英文单词 | P0 |
| 智能批改 | 自动比对标准答案 | P0 |
| 容错机制 | 忽略大小写、常见拼写错误提示 | P1 |
| 错误标记 | 红叉标记错误，显示正确答案 | P0 |
| 成绩统计 | 正确率、用时、错误分布 | P0 |
| 错题本 | 自动收录错误单词 | P0 |
| 历史记录 | 查看历次听写成绩 | P1 |
| 报告分享 | 生成图片/PDF报告分享 | P2 |
| 批量批改 | 一次批改多张听写纸 | P2 |

### 3.3 辅助功能

## 4. 技术方案

### 4.1 技术选型

#### 方案 A：跨平台移动应用 (推荐)

| 层级 | 技术选型 | 说明 |
|-----|---------|-----|
| 前端框架 | **Flutter** / React Native | 一套代码支持 iOS/Android |
| 状态管理 | Riverpod / Provider | 响应式状态管理 |
| 本地存储 | Hive / SQLite | 词库和记录本地存储 |
| OCR引擎 | **PaddleOCR** / ML Kit | 中英文OCR，支持手写体 |
| TTS引擎 | **系统TTS** / Azure TTS | 免费+高质量语音可选 |
| 图像处理 | OpenCV | 图像预处理和增强 |

#### 方案 B：Web + PWA

| 层级 | 技术选型 | 说明 |
|-----|---------|-----|
| 前端框架 | **Next.js** / Vue 3 | 现代Web框架 |
| OCR引擎 | Tesseract.js / 云端API | 浏览器端OCR |
| TTS引擎 | Web Speech API | 浏览器原生支持 |
| 后端服务 | Node.js / Python | 可选，用于高级功能 |

#### 方案 C：桌面应用

| 层级 | 技术选型 | 说明 |
|-----|---------|-----|
| 框架 | **Tauri** / Electron | 跨平台桌面应用 |
| OCR引擎 | PaddleOCR / EasyOCR | 本地OCR |
| TTS引擎 | edge-tts / pyttsx3 | 本地语音合成 |

### 4.2 OCR技术细节

**识别流程**：
```
图片输入 → 图像预处理（去噪/纠偏/二值化）→ 文字检测 → 文字识别 → 后处理 → 结果输出
```

**技术选择**：
- **PaddleOCR**：开源、支持中英文、模型轻量、识别准确率高
- **ML Kit** (Google)：移动端优化、离线识别、集成简单
- **手写识别优化**：针对英文手写体单独训练或使用专门模型

### 4.3 TTS技术细节

**语音合成选项**：
- **系统TTS**：免费、无需网络、音质一般
- **Azure TTS**：高质量、多种语音可选、需网络/API密钥
- **Edge TTS**：免费、高质量、基于Edge浏览器语音
- **Kokoro-TTS**：开源、轻量、本地运行

## 6. 非功能需求

### 6.1 性能要求

| 指标 | 目标值 | 说明 |
|-----|-------|-----|
| APP启动时间 | ≤ 2秒 | 冷启动到可交互 |
| OCR识别速度 | ≤ 3秒 | 单张A4单词表 |
| TTS响应延迟 | ≤ 500ms | 点击播放到出声 |
| 拍照批改速度 | ≤ 5秒 | 单张听写纸 |
| 内存占用 | ≤ 200MB | 运行时峰值 |
| 安装包大小 | ≤ 100MB | Android/iOS |

### 6.2 兼容性要求

| 平台 | 最低版本 | 说明 |
|-----|---------|-----|
| Android | 8.0 (API 26) | 覆盖90%+设备 |
| iOS | 14.0 | 覆盖95%+设备 |
| 屏幕适配 | 4.7寸 - 12.9寸 | 手机+平板 |

### 6.3 安全与隐私

- **本地优先**：词库、听写记录默认本地存储
- **权限最小化**：仅请求相机、存储、麦克风（如需要录音）权限
- **数据加密**：敏感数据本地加密存储
- **无强制账号**：核心功能无需注册登录

## 7. 里程碑规划

### Phase 1: MVP (4-6周)
- 基础UI框架搭建
- 拍照上传 + 基础OCR
- 简单听写功能（英译中）
- 本地词库存储

### Phase 2: 核心功能完善 (4周)
- 听写设置（语速、间隔、重复）
- 中译英模式
- 拍照批改基础版
- 错题本功能

### Phase 3: 体验优化 (4周)
- OCR准确度优化
- 手写识别增强
- 内置词库
- 学习统计

### Phase 4: 高级功能 (待定)
- 云同步
- 社交分享
- AI智能推荐
- 多语言支持

## 8. 风险评估

| 风险 | 可能性 | 影响 | 应对措施 |
|-----|-------|-----|---------|
| OCR手写识别准确率低 | 中 | 高 | 选用成熟模型+用户校正机制 |
| 多平台适配成本 | 中 | 中 | 选用跨平台框架，优先Android |
| TTS语音质量差 | 低 | 中 | 集成多种TTS引擎供用户选择 |
| 用户隐私顾虑 | 中 | 高 | 本地优先策略，明确隐私政策 |
